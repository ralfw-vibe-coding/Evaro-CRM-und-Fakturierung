CI-Grundfarbe der Anwendung: #F60B8A, Schrift soll natürlich schwarz sein.

Der Header der App zeigt den App-Namen und den Claim oben links.
Rechts im Header gibt es ein User Icon mit dem Kürzel des Users und mit einem User Menü.
Das User Menü bietet:

- Profil bearbeiten
- Abmelden

Außerdem kann man im Header den Bereich der App wählen: CRM bzw. Fakturierung.
Damit steht der Platz unter dem Header ganz dem jeweiligen Bereich zur Verfügung.

# Bereich CRM

Im CRM steht die Übersicht über Konakte im Vordergrund. Meistens werden Kontakte gesucht. Die Angaben zu einem Kontakt sind überschaubar. Sie passen auf eine Karte. Kontakte sollen also vor allem als Karten in Anlehnung an Visitenkarten dargestellt werden.

Da das Filtern von Kontakten so wichtig ist und Filter gespeichert werden sollen, braucht es dafür Platz.

Die Aufteilung des CRM Bereichs scheint deshalb wie folg sinnvoll:

- links eine Spalte, um zu Filtern.
- in der Mitte die Anzeige der gefilterten Konakte; die Anzeige kann auch mehrere Kontaktkarten nebeneinander anzeigen
- rechts die Details eines selektierten Kontaktes

Filter muss man speichern, aus einer Liste auswählen und löschen können.

Kontakte muss man auswählen können für die Detailansicht, löschen können und neu anlegen.

Kontakte neu anlegen, soll auf zwei Weisen möglich sein:

- strukturiert, d.h. in dem man ein Formular ausfüllt, in dem man die Kontaktdetails und ggf. Geschäftspartner hinzufügt
- unstrukturiert, d.h. indem man zb einen Email Text einsetzt, aus dem automatisch Kontaktdetails extrahiert werden (die man dann nur noch überprüft)

Die Speicherung von Änderungen soll immer explizit stattfinden, also indem der User einen Speichern-Button klickt.

## Interaktionen

Es liegen einige Interaktionen mit zugehörigen Requests/Responses (Commands bzw. Queries) auf der Hand:

## User Management / Auth

- Login angefordert
- OTP eingegeben um Login abzuschließen
- User-Profildaten speichern (nach Veränderung)
- Logout

## Kontakte

- Kontakt und/oder Geschäftspartner nach Ersterfassung oder Änderung speichern
- Kontakt/Geschäftspartner löschen
- Kontakte filtern (bzw. selektieren)

## Filter

Filter sind userspezifisch. Sie sollen Usern von Gerät zu Gerät folgen.

- Filter speichern (erstmalig oder nach Änderung)
- Alle Filter laden
- Filter löschen
